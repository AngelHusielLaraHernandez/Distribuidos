    \section{Objetivo:}

    Construir un sistema distribuido de cómputo de alto rendimiento (HPC) y ejecutar programas paralelos que distribuyan el trabajo entre los nodos que lo conforman.

    \section{Desarrollo:}
    Para realizacion del proyecto se tiene como funciones disponibles de MPI las siguientes:
    \begin{itemize}
        \item \textbf{MPI\_Init:} Inicializa el entorno de ejecución de MPI.
        \item \textbf{MPI\_Comm\_size:} Determina el número total de procesos en el comunicador.
        \item \textbf{MPI\_Comm\_rank:} Determina el identificador único (rango) del proceso dentro del comunicador.
        \item \textbf{MPI\_Send:} Envía un mensaje a otro proceso.
        \item \textbf{MPI\_Recv:} Recibe un mensaje de otro proceso.
        \item \textbf{MPI\_Finalize:} Finaliza el entorno de ejecución de MPI.
        \item \textbf{MPI\_Gather:} Recopila datos de todos los procesos en el comunicador.
        \item \textbf{MPI\_Scatter:} Distribuye datos desde un proceso raíz a todos los procesos en el comunicador.
        \item \textbf{MPI\_Bcast:} Difunde un mensaje desde un proceso raíz a todos los procesos en el comunicador.
        \item \textbf{MPI\_Reduce:} Realiza una operación de reducción (como suma, máximo, etc.) en datos distribuidos entre los procesos y devuelve el resultado al proceso raíz.
    \end{itemize}

    De las cuales, para el desarrollo de los programas, solo se usaron las funciones \textbf{MPI\_Init}, \textbf{MPI\_Comm\_size}, \textbf{MPI\_Comm\_rank}, \textbf{MPI\_Send}, \textbf{MPI\_Recv}, \textbf{MPI\_Scatter}, \textbf{MPI\_Gather} y \textbf{MPI\_Finalize}.

El código es el siguiente:

\begin{lstlisting}[style=cstyle,caption={Distribucion de datos desde el proceso maestro a los nodos clientes con MPI},label={lst:programa-distribucion}]
#include <mpi.h>
#include <stdio.h>
#include <string.h>

int main(int argc, char** argv) {
    int rank, size;
    char processor_name[MPI_MAX_PROCESSOR_NAME];
    int name_len;

    // Inicializar MPI
    MPI_Init(&argc, &argv);
    
    MPI_Comm_rank(MPI_COMM_WORLD, &rank);
    MPI_Comm_size(MPI_COMM_WORLD, &size);
    MPI_Get_processor_name(processor_name, &name_len);

    // Verificacion de seguridad
    if (size < 6) {
        if (rank == 0) {
            printf("Error: Se necesitan 6 procesos para los 6 nodos.\n");
        }
        MPI_Finalize();
        return 0;
    }

    if (rank == 0) {
        // Logica del Servidor (Proceso 0)
        printf("Soy el maestro (Proceso 0) en el nodo: %s\n", processor_name);
        printf("Enviando un dato unico a los 5 nodos clientes...\n\n");
        
        // Enviar a los procesos del 1 al 5
        for (int dest = 1; dest <= 5; dest++) {
            int dato_a_enviar = dest * 100; // Datos: 100, 200, 300, 400, 500
            MPI_Send(&dato_a_enviar, 1, MPI_INT, dest, 0, MPI_COMM_WORLD);
        }
    } else if (rank >= 1 && rank <= 5) {
        // Logica de los Clientes (Procesos 1 al 5)
        int dato_recibido;
        
        MPI_Recv(&dato_recibido, 1, MPI_INT, 0, 0, MPI_COMM_WORLD, MPI_STATUS_IGNORE);
        
        printf("Nodo cliente: %s | ID de proceso: %d | Dato recibido: %d\n", 
               processor_name, rank, dato_recibido);
    }

    // Finalizar MPI
    MPI_Finalize();
    return 0;
}
\end{lstlisting}
    

Este programa escrito en C utiliza la biblioteca MPI (\textit{Message Passing Interface}) para comunicar diferentes procesos que pueden estar ejecutándose en distintas computadoras o núcleos (nodos). El código sigue un modelo clásico de ``maestro-cliente''. A continuación, se detalla su funcionamiento paso a paso:

\begin{itemize}
    \item \textbf{Preparación inicial:} 
    Las funciones \texttt{MPI\_Init}, \texttt{MPI\_Comm\_rank} y \texttt{MPI\_Comm\_size} se encargan de encender el sistema de comunicación. A cada proceso se le asigna un número identificador único llamado \texttt{rank} (el proceso 0 será nuestro maestro, y los demás serán los clientes). También se cuenta cuántos procesos hay en total (\texttt{size}) y se guarda el nombre del equipo donde se está ejecutando mediante \texttt{MPI\_Get\_processor\_name}.

    \item \textbf{Verificación de seguridad:}
    El programa comprueba mediante un \texttt{if (size < 6)} que se hayan lanzado al menos 6 procesos en total (1 maestro y 5 clientes). Si el usuario ejecutó el programa con menos procesos, el maestro muestra un mensaje de error y el programa se detiene de forma segura.

    \item \textbf{La labor del Maestro (Proceso 0):}
    Si el \texttt{rank} es 0, este proceso toma el rol de jefe. Su trabajo es usar un bucle \texttt{for} para enviar un número matemático simple y único a cada cliente. Usando la instrucción \texttt{MPI\_Send}, le envía un 100 al proceso 1, un 200 al proceso 2, un 300 al proceso 3, y así sucesivamente.

    \item \textbf{La labor de los Clientes (Procesos del 1 al 5):}
    Si el \texttt{rank} está entre el 1 y el 5, el proceso actúa como cliente. Su única tarea es esperar a que el maestro le envíe la información utilizando la instrucción \texttt{MPI\_Recv}. Una vez que el dato llega, el cliente imprime en la pantalla dónde se está ejecutando, cuál es su número de proceso y qué número recibió exactamente.

    \item \textbf{Cierre del programa:}
    Al final, la instrucción \texttt{MPI\_Finalize()} se asegura de cerrar las conexiones de red y apagar el entorno MPI de manera limpia para no dejar procesos atascados en la memoria.
\end{itemize}

    \textbf{Enunciado: }

Además, se reserva la memoria dinámica para los punteros de las matrices enteras $A$ y $B$, las cuales tendrán un tamaño total de $N \times M$.
    
    Luego se inicializa el entorno de ejecución de MPI con \texttt{MPI\_Init()}, se obtiene el número total de procesos con \texttt{MPI\_Comm\_size()} y el identificador único de cada proceso con \texttt{MPI\_Comm\_rank()}. También se obtiene el nombre del nodo físico utilizando la función \texttt{gethostname()}. Se implementa una validación para asegurar que existan al menos 2 procesos, ya que es el mínimo indispensable para conformar la topología de anillo.

    Esta función recibe el título de la matriz, el puntero a los datos y sus dimensiones, iterando mediante un ciclo anidado para mostrar los elementos con un formato de espaciado uniforme.

    Posteriormente, comienza la lógica específica del proceso raíz:
    
    \textbf{Fragmento de codigo de generacion de matriz (C):}

    \begin{lstlisting}[style=cstyle,caption={Generacion de matrices e inicio del anillo en proceso 0},label={lst:programa-a}]
    // --- LOGICA DEL PROCESO 0 (Inicia y Termina el ciclo) ---
    if (id == 0) {
        // Inicializar matrices A y B
        for (int i = 0; i < N * M; i++) {
            A[i] = i + 1; // Valores secuenciales
            B[i] = 2;     // Valor constante para que sea facil verificar la suma
        }

        printf("=========================================================\n");
        printf("  INICIO DEL PROGRAMA - %d procesos\n", np);
        printf("=========================================================\n");
        printf("  [P0] Nodo: %s | Genera e imprime matrices iniciales\n\n", hostname);
        print_matrix("Matriz A (Inicial)", A, N, M);
        printf("\n");
        print_matrix("Matriz B (Constante)", B, N, M);
        printf("=========================================================\n");

        // Enviar A y B al Proceso 1
        printf("  [P0] Nodo: %s | Enviando matrices a P1...\n", hostname);
        MPI_Send(A, N * M, MPI_INT, 1, 0, MPI_COMM_WORLD);
        MPI_Send(B, N * M, MPI_INT, 1, 1, MPI_COMM_WORLD);
    \end{lstlisting}

    Cuando la condición evalúa al proceso 0, este se encarga de inicializar la matriz $A$ con valores numéricos secuenciales y la matriz $B$ con un valor constante (2) para facilitar la verificación del cálculo distribuido. Tras imprimir los valores iniciales en consola, el proceso 0 envía ambas matrices completas al Proceso 1 utilizando la función \texttt{MPI\_Send()}. 
    
    El primer \texttt{MPI\_Send()} envía la matriz $A$, especificando su puntero, la cantidad total de elementos ($N \times M$), el tipo de dato (\texttt{MPI\_INT}), el proceso destino (1), una etiqueta de mensaje (0) y el comunicador. Inmediatamente después, envía la matriz $B$ utilizando la etiqueta de mensaje (1).

    Al mismo tiempo, el resto de los procesos que conforman el anillo ejecutan la lógica de recepción, suma y reenvío:

    \begin{lstlisting}[style=cstyle,caption={Recepcion, suma local y reenvio en procesos trabajadores},label={lst:programa-a}]
    // --- LOGICA DE LOS PROCESOS TRABAJADORES (1 a np-1) ---
    else {
        // Recibir del proceso anterior (id - 1)
        MPI_Recv(A, N * M, MPI_INT, id - 1, 0, MPI_COMM_WORLD, MPI_STATUS_IGNORE);
        MPI_Recv(B, N * M, MPI_INT, id - 1, 1, MPI_COMM_WORLD, MPI_STATUS_IGNORE);

        printf("  [P%d] Nodo: %s | Recibio matrices de P%d. Sumando A = A + B...\n", id, hostname, id - 1);

        // Sumar ambas matrices y almacenar en A
        for (int i = 0; i < N * M; i++) {
            A[i] = A[i] + B[i];
        }

        // Determinar el siguiente nodo (si es el ultimo, envia al 0)
        int next_node = (id + 1) % np;
        
        // Enviar al siguiente nodo
        MPI_Send(A, N * M, MPI_INT, next_node, 0, MPI_COMM_WORLD);
        MPI_Send(B, N * M, MPI_INT, next_node, 1, MPI_COMM_WORLD);
        printf("  [P%d] Nodo: %s | Matrices enviadas a P%d.\n", id, hostname, next_node);
    }
    \end{lstlisting}

    En este bloque, cada proceso intermedio utiliza \texttt{MPI\_Recv()} para capturar las matrices enviadas por su predecesor (\texttt{id - 1}). Una vez recibidos los datos, se ejecuta un ciclo \texttt{for} lineal que recorre los arreglos unidimensionales para sumar cada celda de la matriz $B$ a su correspondiente en la matriz $A$, sobreescribiendo el valor directamente en $A$. 
    
    Para continuar con la topología de anillo, se calcula la identidad del siguiente proceso destino mediante la operación de módulo aritmético \texttt{next\_node = (id + 1) \% np}. Esta fórmula garantiza que el último proceso apunte de vuelta al proceso 0. Finalmente, se despachan los datos al nodo calculado.

    Por último, el programa se cierra de regreso en el flujo del proceso 0, el cual se había quedado bloqueado esperando la conclusión del ciclo:

    \begin{lstlisting}[style=cstyle,caption={Recepcion de resultados y finalizacion en proceso 0},label={lst:programa-a}]
        // Esperar a recibir el resultado del ultimo proceso
        MPI_Recv(A, N * M, MPI_INT, np - 1, 0, MPI_COMM_WORLD, MPI_STATUS_IGNORE);
        MPI_Recv(B, N * M, MPI_INT, np - 1, 1, MPI_COMM_WORLD, MPI_STATUS_IGNORE);

        // Imprimir resultado final
        printf("\n");
        printf("=========================================================\n");
        printf("  RESULTADO FINAL - Ciclo completado\n");
        printf("=========================================================\n");
        printf("  [P0] Nodo: %s | Recibio resultado de P%d\n\n", hostname, np - 1);
        print_matrix("Matriz A (Final despues de todas las sumas)", A, N, M);
        printf("=========================================================\n");

    } 
    
    free(A);
    free(B);
    MPI_Finalize();
    return 0;
    }
    \end{lstlisting}

    El proceso 0 emplea llamadas a \texttt{MPI\_Recv()} indicando explícitamente que espera el mensaje proveniente del último nodo del clúster (\texttt{np - 1}). Al recibir las matrices modificadas tras pasar por todos los procesos trabajadores, el proceso 0 imprime la matriz $A$ resultante que contiene la sumatoria global acumulada. Para finalizar, se libera la memoria reservada para ambas matrices mediante \texttt{free()} y se cierra de forma segura el entorno de paso de mensajes con \texttt{MPI\_Finalize()}.

    la cual se encarga de enviar partes de la matriz a cada proceso, para esto los elementos de esta funcion se componen:

    \begin{lstlisting}[style=cstyle,caption={Distribucion de matriz con MPI\_Scatter},label={lst:programa-b}]
        local_A = (int *)malloc(rows_per_proc * M * sizeof(int));
        MPI_Scatter(A, rows_per_proc * M, MPI_INT, local_A, rows_per_proc * M, MPI_INT, 0, MPI_COMM_WORLD);
    \end{lstlisting}
   
    Donde el primer argumento es el puntero a la matriz $A$ que se va a distribuir, el segundo argumento es el número de elementos que cada proceso recibirá (en este caso, el número de filas por proceso multiplicado por el número de columnas), el tercer argumento especifica el tipo de dato (en este caso, enteros), el cuarto argumento es un puntero al bloque local donde cada proceso almacenará su parte de la matriz, el quinto argumento es nuevamente el número de elementos que cada proceso recibirá, el sexto argumento especifica el tipo de dato para la recepción, el séptimo argumento indica el proceso raíz (en este caso, 0) y el octavo argumento es el comunicador.
    
    Ahora vien sepues de estos e empieza con el calculo de la suma, para esto primero inicializaciamos las variables \texttt{fila\_global\_inicio = id * rows\_per\_proc;}, la cual se encarga de calcular el índice de la primera fila que cada proceso analizará, 
    tambien tenemos a las variables \texttt{local\_max\_sum} y \texttt{local\_max\_row}, las cuales se encargan de almacenar la suma máxima local y el índice de la fila con la suma máxima local, respectivamente.

    Despues de esto se imprime que procesos recibio que pila en especifico, donde la linea de codigo es la siguiente:

    \begin{lstlisting}[style=cstyle,caption={Impresion de bloque local recibido por cada proceso},label={lst:programa-b}]
        printf("\n  [P%d] Nodo: %s | Recibio filas %d a %d\n",
           id, hostname, fila_global_inicio, fila_global_inicio + rows_per_proc - 1);
    \end{lstlisting}

    Luego de esto, ahora los procesos lo que haran sera entrar en ciclo for para calcular la suma de cada fila dentro de su bloque local, y determinar la fila con la suma máxima local.
    
    Para esto, el bloque de codigo es el siguiente:

    \begin{lstlisting}[style=cstyle,caption={Calculo de suma de filas y determinacion de fila con suma maxima local},label={lst:programa-b}]
        for (int i = 0; i < rows_per_proc; i++) {
            int sum = 0;
            int fila_global = fila_global_inicio + i;

            printf("  [P%d] Nodo: %s | Sumando fila %d: ", id, hostname, fila_global);
            for (int j = 0; j < M; j++) {
                sum += local_A[i * M + j];
                if (j > 0) printf(" + ");
                printf("%d", local_A[i * M + j]);
            }
            printf(" = %d\n", sum);

            if (i == 0 || sum > local_max_sum) {
                local_max_sum = sum;
                local_max_row = i;
            }
        }
    \end{lstlisting}

    En este bloque de codigo, cada proceso recorre las filas de su bloque local, calcula la suma de los elementos de cada fila y determina cuál es la fila con la suma máxima local.
    Para esto, se utiliza un ciclo for para iterar sobre las filas del bloque local, y dentro de este ciclo, se calcula la suma de los elementos de cada fila utilizando otro ciclo for anidado.

    La estructura de funcionamiento del ciclo for anidado, es por medio de un contador de filas locales, el cual se encarga de recorrer las filas del bloque local, y dentro de este ciclo, se tiene otro ciclo for que recorre las columnas de la fila actual para calcular la suma de los elementos de esa fila.
    
    Ademas, se imprime el proceso que esta realizando la suma, el nodo en el que se encuentra, la fila que esta sumando y los elementos de esa fila, junto con el resultado de la suma.
    Luego de esto, se compara la suma calculada con la suma máxima local almacenada en la variable \texttt{local\_max\_sum}, y si la suma actual es mayor, se actualiza la suma máxima local y el índice de la fila con la suma máxima local.
    
    Siguiendo con las demas linea de codigo, tenemos la inicializacion de la variables \texttt{global\_max\_row = fila\_global\_inicio + local\_max\_row;}, donde se calcula el índice global de la fila con la suma máxima local, sumando el índice de la primera fila del bloque local con el índice de la fila con la suma máxima local dentro de ese bloque.
    
    Luego de esto, se imprime el resultado local de cada proceso, indicando cuál es la fila con la suma máxima local y el valor de esa suma.

    \begin{lstlisting}[style=cstyle,caption={Impresion de resultado local de cada proceso},label={lst:programa-b}]
        int global_max_row = fila_global_inicio + local_max_row;

        printf("  [P%d] Nodo: %s | Mayor suma local -> Fila %d (suma = %d)\n",
            id, hostname, global_max_row, local_max_sum);
        separador();
    \end{lstlisting}

    Despues de esto, el proceso 0 se encarga de reservar memoria para almacenar los resultados locales de cada proceso, que consisten en el índice global de la fila con la suma máxima local y el valor de esa suma máxima local.
    Para esto, el bloque de codigo es el siguiente:

    \begin{lstlisting}[style=cstyle,caption={Reserva de memoria para resultados locales en proceso 0},label={lst:programa-b}]
        int *local_max_rows = NULL;
        int *local_max_sums = NULL;
        if (id == 0) {
            local_max_rows = (int *)malloc(np * sizeof(int));
            local_max_sums = (int *)malloc(np * sizeof(int));
        }
    \end{lstlisting}

    En seguida, se utiliza la función \texttt{MPI\_Gather()} para recopilar los resultados locales de cada proceso en el proceso 0.
    
    Para esto, el bloque de codigo es el siguiente:

    \begin{lstlisting}[style=cstyle,caption={Recopilacion de resultados locales con MPI\_Gather},label={lst:programa-b}]
        MPI_Gather(&global_max_row, 1, MPI_INT, local_max_rows, 1, MPI_INT, 0, MPI_COMM_WORLD);
        MPI_Gather(&local_max_sum, 1, MPI_INT, local_max_sums, 1, MPI_INT, 0, MPI_COMM_WORLD);
    \end{lstlisting}

    Para la primera linea de \texttt{MPI\_Gather()}, tenemos que el primer argumento es un puntero al dato local que cada proceso va a enviar (en este caso, el índice global de la fila con la suma máxima local), el segundo argumento es el número de elementos que cada proceso enviará (en este caso, 1), el tercer argumento especifica el tipo de dato (en este caso, enteros), el cuarto argumento es un puntero al arreglo donde el proceso raíz almacenará los datos recopilados (en este caso, \texttt{local\_max\_rows}), el quinto argumento es nuevamente el número de elementos que cada proceso enviará, el sexto argumento especifica el tipo de dato para la recepción, el séptimo argumento indica el proceso raíz (en este caso, 0) y el octavo argumento es el comunicador.
    La segunda linea de \texttt{MPI\_Gather()} es similar, pero en este caso se recopilan los valores de las sumas máximas locales en el arreglo \texttt{local\_max\_sums}.

    Por ultimo tenemos el final del programa, donde por medio de una condicional se veririfica que el que imprima el resultado y calcula la ultima parte sea el procesos 0, donde al entrar a la conficional, el proceso 0 imrpimira los resiltados e imprimira un anuncio que el proceo 0 recolecto los resultados
    y luego de esto, se entra en un ciclo for para determinar cuál es la fila con la suma máxima global, comparando las sumas máximas locales recopiladas y sus correspondientes índices de fila.

    Finalmente, el proceso 0 imprime el resultado final, indicando cuál es la fila con la suma máxima global y el valor de esa suma, esto lo hace por medio de la siguiente linea de codigo:

    \begin{lstlisting}[style=cstyle,caption={Impresion de resultado final en proceso 0},label={lst:programa-b}]
        if (id == 0) {
            printf("\n");
            separador();
            printf("  RESULTADO FINAL\n");
            separador();
            printf("  [P0] Nodo: %s | Recolecto resultados de %d procesos (MPI_Gather)\n\n", hostname, np);

            int best_sum = all_sums[0];
            int best_row = all_rows[0];

            for (int i = 0; i < np; i++) {
                printf("  -> P%d reporta: Fila %d con suma %d", i, all_rows[i], all_sums[i]);
                if (i == 0 || all_sums[i] > best_sum) {
                    if (i > 0) {
                        best_sum = all_sums[i];
                        best_row = all_rows[i];
                    }
                }
                printf("\n");
            }

            printf("\n  >>> La fila con mayor suma es la FILA %d con suma = %d <<<\n", best_row, best_sum);
            separador();

            free(all_sums);
            free(all_rows);
        }
    \end{lstlisting}

    La conficional dentro del ciclo for se encarga de comparar las sumas máximas locales recopiladas y actualizar la suma máxima global y el índice de la fila con la suma máxima global si se encuentra una suma mayor, el orden de impresion es por medio de un ciclo for que recorre los resultados recopilados y los imprime, indicando qué proceso reportó cada fila y su suma correspondiente. Finalmente, se imprime el resultado final indicando cuál es la fila con la suma máxima global y el valor de esa suma, esta se almaceno en las variables \texttt{best\_row} y \texttt{best\_sum} respectivamente. Finalmente el proceso 0 libera la memoria asignada para los arreglos de resultados locales.
